## How Authentication Works üîê

**Current Implementation:**
- localStorage-based (demo/prototype)
- Stores: name, email, password
- Session persists across page reloads
- Sign up creates new user
- Sign in validates credentials
- Sign out clears session

**Important:** This is for demonstration only!

**For Production:**
You'd need:
- Real database (PostgreSQL, MongoDB, Supabase)
- Password hashing (bcrypt)
- JWT tokens or sessions
- Backend API
- Email verification
- Password reset flow
- OAuth (Google, GitHub)
